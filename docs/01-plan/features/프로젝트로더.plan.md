## 1. 현재 상태 분석

### 1.1 하드코딩된 구조

```
CONFIG (line ~2384)
├── TOTAL_BUDGET: 70000000
├── WITHHOLDING_THRESHOLD: 125000
├── PERIODS: { h1, h2 }

DEFAULT_ORG_NAME (line ~2395): '청년노동자인권센터'

BUDGET_DATA (line ~2934, ~140줄)
├── projectName: "2026 공익단체 인큐베이팅 지원사업"
├── organization: "청년노동자인권센터"
├── totalBudget: 70000000
├── period: "2026.01 ~ 2026.12"
├── periods: { total, h1, h2 }
└── categories: [
    ├── { id: "project-costs", name: "사업비", subcategories: [...] }
    └── { id: "operating-costs", name: "운영비", subcategories: [...] }
]
```

### 1.2 문제점

- BUDGET_DATA가 `<script>` 블록에 직접 정의 → 년도 변경 시 코드 수정 필수
- CONFIG.TOTAL_BUDGET, DEFAULT_ORG_NAME도 하드코딩
- budget_executions 테이블에 project_id 개념 없음 → 모든 집행이 단일 프로젝트
- 다른 단체/년도 데이터를 함께 볼 수 없음

## 2. 구현 계획

### 2.1 데이터 모델

#### Supabase 신규 테이블: `projects`

```sql
CREATE TABLE IF NOT EXISTS public.projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,           -- "2026 공익단체 인큐베이팅 지원사업"
    organization VARCHAR(100) NOT NULL,   -- "청년노동자인권센터"
    year INTEGER NOT NULL,                -- 2026
    period VARCHAR(50),                   -- "2026.01 ~ 2026.12"
    total_budget INTEGER NOT NULL,        -- 70000000
    budget_data JSONB NOT NULL,           -- BUDGET_DATA 전체 구조
    config JSONB,                         -- CONFIG (세율, 기준액 등)
    modules JSONB DEFAULT '["dashboard","budget","schedule","board","gallery","newsletter","guide"]',
    module_config JSONB DEFAULT '{}',     -- 커스텀 모듈 설정
    is_active BOOLEAN DEFAULT false,      -- 현재 활성 프로젝트
    created_by UUID REFERENCES public.users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

- `budget_data`: BUDGET_DATA의 categories 배열 전체를 JSONB로 저장
- `config`: withholding_threshold, tax_rate 등 프로젝트별 설정
- `modules`: 활성화된 모듈(페이지) 목록 배열 — 네비게이션에 이 목록만 표시
- `module_config`: 모듈별 추가 설정 (라벨, 아이콘 등)
- `is_active`: 현재 로드된 프로젝트 (하나만 true)

### 2.2 프로젝트별 커스텀 모듈 시스템

#### 문제

현재 학교관리(schools) 페이지는 청년노동자인권센터에만 해당하는 기능.
다른 단체 프로젝트에서는 불필요하거나, 자기만의 특수 페이지가 필요할 수 있음.

#### 설계: modules 배열 기반 네비게이션 제어

```javascript
// 공통 모듈 (모든 프로젝트에 기본 포함)
const COMMON_MODULES = [
  { id: 'dashboard', label: '대시보드', icon: '📊' },
  { id: 'budget',    label: '예산 관리', icon: '💰' },
  { id: 'schedule',  label: '일정 관리', icon: '📅' },
  { id: 'board',     label: '게시판',    icon: '📋' },
  { id: 'gallery',   label: '갤러리',    icon: '🖼️' },
  { id: 'newsletter',label: '뉴스레터',  icon: '📰' },
  { id: 'guide',     label: '회계가이드', icon: '📖' },
];

// 프로젝트 전용 모듈 (프로젝트별 활성화)
const CUSTOM_MODULES = [
  { id: 'schools',   label: '학교관리',  icon: '🏫' },
  // 향후 다른 단체 전용 모듈 추가 가능
];
```

#### 동작 방식

1. **네비게이션 렌더링**: `project.modules` 배열에 포함된 모듈만 메뉴에 표시
2. **프로젝트 생성 시**: 공통 모듈 자동 포함 + 커스텀 모듈 선택 체크박스
3. **기존 코드 호환**: renderSchools() 등 기존 함수는 그대로 유지, 네비게이션에서만 분기

```javascript
// 네비게이션 렌더링 예시
const activeModules = activeProject?.modules || [...COMMON_MODULES.map(m => m.id)];
const allModules = [...COMMON_MODULES, ...CUSTOM_MODULES];
const visibleModules = allModules.filter(m => activeModules.includes(m.id));

// + 관리자는 항상 관리자 탭 표시
```

#### 프로젝트별 설정 예시

```json
// 청년노동자인권센터 2026
{
  "modules": ["dashboard", "budget", "schedule", "board", "gallery",
              "newsletter", "schools", "guide"],
  "module_config": {
    "schools": { "label": "학교관리", "icon": "🏫",
                 "description": "파트너 학교 및 시간표 관리" }
  }
}

// 다른 단체 A (환경 단체)
{
  "modules": ["dashboard", "budget", "schedule", "board", "gallery",
              "newsletter", "guide"],
  "module_config": {}
}
```

#### 향후 확장성

- 새로운 단체 전용 기능은 renderXxx() 함수로 구현
- CUSTOM_MODULES에 등록만 하면 프로젝트 생성 시 선택 가능
- module_config로 모듈별 세부 설정 전달 (API 키, 외부 서비스 연동 등)

### 2.2 기능 구조

#### A. 프로젝트 목록 관리 (관리자 페이지)

```
관리자 → 프로젝트 관리 탭
├── 현재 활성 프로젝트 표시 (하이라이트)
├── 프로젝트 목록 (년도/단체별 필터)
│   ├── 프로젝트명, 단체명, 년도, 총예산, 상태
│   └── [활성화] [수정] [삭제] 버튼
├── 새 프로젝트 생성
│   ├── 방법 1: 엑셀 업로드 (사업계획서)
│   ├── 방법 2: 기존 프로젝트 복제 (년도만 변경)
│   └── 방법 3: JSON 직접 입력
└── 프로젝트 전환 시 확인 모달
```

#### B. 엑셀 사업계획서 파싱

은행 엑셀과 달리, 사업계획서 엑셀은 예산편성표 형태:

```
| 대분류 | 소분류 | 항목명 | 유형 | 총예산 | 상반기 | 하반기 | 산출내역 |
|--------|--------|--------|------|--------|--------|--------|----------|
| 사업비 | 교과서연구 | 세미나비 | 사업인건비 | 1,000,000 | 500,000 | 500,000 | 4회*250,000 |
```

파싱 플로우:
1. 엑셀 업로드 → SheetJS로 파싱
2. 헤더 자동감지 (대분류/소분류/항목명/금액 등)
3. 트리 구조 변환: 행 → categories[].subcategories[].items[]
4. 미리보기 표시 (트리뷰)
5. 프로젝트 메타 입력 (이름, 단체, 년도)
6. 저장 → projects 테이블 INSERT

#### C. 프로젝트 로딩 (활성화)

1. 프로젝트 선택 → `is_active = true` 설정 (다른 건 false)
2. `projects` 테이블에서 `budget_data` JSONB 로드
3. 앱 상태에 반영:
   - BUDGET_DATA → 프로젝트의 budget_data
   - CONFIG.TOTAL_BUDGET → 프로젝트의 total_budget
   - DEFAULT_ORG_NAME → 프로젝트의 organization
4. 기존 하드코딩 BUDGET_DATA는 기본값(fallback)으로 유지

### 2.4 UI 배치

```
관리자 페이지 (renderAdmin)
├── 기존: 사용자 관리, 수급자 관리, 기관 설정
└── 추가: 📂 프로젝트 관리 탭
    ├── 현재 활성 프로젝트 정보 카드
    ├── 프로젝트 목록 (년도/단체 필터)
    ├── 새 프로젝트 생성
    │   ├── 기본 정보 입력 (이름, 단체, 년도, 기간)
    │   ├── 엑셀 업로드 또는 기존 복제
    │   └── 모듈 선택 체크박스 (공통 + 커스텀)
    └── 프로젝트 전환 확인 모달
```

### 2.5 구현 우선순위

| 순서 | 항목 | 설명 |
|------|------|------|
| 1 | DB 마이그레이션 | projects 테이블 (modules, module_config 포함) + 현재 데이터 마이그레이션 |
| 2 | 프로젝트 목록 UI | 관리자 페이지에 프로젝트 관리 탭 |
| 3 | 프로젝트 로딩 로직 | 활성 프로젝트 로드 → BUDGET_DATA + 네비게이션 동적 반영 |
| 4 | 모듈 시스템 | 네비게이션에 modules 배열 기반 표시/숨김 |
| 5 | 엑셀 업로드 파싱 | 사업계획서 엑셀 → BUDGET_DATA 변환 |
| 6 | 프로젝트 복제 | 기존 프로젝트 기반 새 년도 생성 |

## 3. 제약사항 및 리스크

| 항목 | 설명 | 대응 |
|------|------|------|
| budget_executions에 project_id 없음 | 기존 집행 데이터와 프로젝트 연결 필요 | 마이그레이션 시 기본 프로젝트에 귀속 |
| 엑셀 양식 다양성 | 단체마다 사업계획서 양식이 다름 | 열 매핑 UI + 자동감지 |
| 단일 사용자 특성 | 동시 접속 없음 → is_active 충돌 없음 | 단순 boolean으로 충분 |
| 하드코딩 참조 | BUDGET_DATA를 직접 참조하는 코드 다수 | 동적 로딩 후 동일 변수에 할당하여 호환성 유지 |

## 4. 기술 결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 저장소 | Supabase JSONB | 유연한 구조, 스키마 변경 불필요 |
| 파싱 | SheetJS (이미 CDN 로드됨) | 추가 의존성 없음 |
| 상태 관리 | React useState | 기존 패턴 유지 |
| 호환성 | BUDGET_DATA 변수 재할당 | 기존 코드 수정 최소화 |
| 프로젝트 전환 | 페이지 리로드 없이 상태 교체 | UX 개선 |

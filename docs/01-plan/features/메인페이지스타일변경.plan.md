# Plan: 메인페이지(대시보드) 스타일 변경

## Executive Summary

| 항목 | 내용 |
|------|------|
| **Feature** | 메인페이지스타일변경 |
| **시작일** | 2026-03-22 |
| **예상 기간** | 1일 |

| 관점 | 설명 |
|------|------|
| **Problem** | 현재 대시보드가 기본 카드 스타일로 되어 있어 시각적 임팩트가 부족하고, 정보 밀도가 낮아 한눈에 현황을 파악하기 어려움 |
| **Solution** | 모던한 그라데이션 헤더, 글래스모피즘 카드, 개선된 통계 레이아웃, 정보 계층 구조 재설계 |
| **Function UX Effect** | 대시보드 진입 시 사업 전체 현황을 3초 내 파악 가능, 시각적 완성도 향상으로 사용 만족도 증가 |
| **Core Value** | 사업 관리자가 매일 확인하는 메인 화면의 가독성과 심미성을 높여 업무 효율 향상 |

## 1. 배경 및 목적

### 1.1 현재 상태
- 대시보드(`renderDashboard`)는 기본 `.card`, `.stat-card`, `.stats-grid` 스타일 사용
- 배경색 `#F8F6F1`, 카드 `#FFFFFF` + 테두리 `#EBE8E1`
- 통계 카드 4개 (예산 집행률, 예정 일정, 갤러리 사진, 게시글) 동일 스타일
- 예산 소진 분석, 성과지표, 월별 차트, 일정/게시글 목록 순서

### 1.2 목표
- **시각적 계층**: 중요 정보(예산 집행률)를 시각적으로 강조
- **모던 UI**: 2026 트렌드 반영 (그라데이션, 미세 그림자, 둥근 모서리)
- **정보 밀도**: 동일 공간에 더 많은 정보를 효과적으로 전달
- **일관성**: 프로젝트 메인 컬러(#134E42) 활용 강화

## 2. 변경 범위

### 2.1 스타일 변경 대상

| 영역 | 현재 | 변경 후 |
|------|------|---------|
| **섹션 타이틀** | 텍스트만 | 아이콘 + 서브텍스트 + 구분선 |
| **통계 카드** | 동일 흰색 배경 4개 | 각 카드별 고유 그라데이션 + 아이콘 강조 |
| **예산 소진 분석** | 단순 프로그레스 바 | 원형 게이지 또는 세미서클 차트 |
| **성과지표** | 회색 배경 박스 | 컬러 보더 + 진행도 인디케이터 |
| **일정/게시글 목록** | 기본 item-card | 타임라인 스타일 또는 컴팩트 리스트 |
| **갤러리 썸네일** | 기본 그리드 | 메이슨리 또는 오버랩 스타일 |
| **전체 여백/간격** | padding 32px 고정 | 반응형 여백 시스템 |

### 2.2 변경하지 않는 것
- 데이터 로직 (Supabase 쿼리, 통계 계산)
- 네비게이션 구조 (currentPage 기반)
- 다른 페이지(예산, 일정, 갤러리 등)의 스타일
- 모바일 반응형 브레이크포인트 구조

## 3. 디자인 방향

### 3.1 컬러 시스템
```
Primary:    #134E42 (다크 틸) — 헤더, 강조
Secondary:  #1B6B5A (미디엄 틸) — 호버, 액센트
Background: #F8F6F1 (베이지) — 유지
Card:       #FFFFFF → 약간의 그라데이션 or 글래스모피즘
Success:    #22C55E (초록)
Warning:    #F59E0B (주황)
Danger:     #EF4444 (빨강)
Info:       #3B82F6 (파랑)
```

### 3.2 스타일 키워드
- **Clean & Minimal**: 불필요한 장식 최소화
- **Data-first**: 숫자와 차트가 가장 먼저 눈에 들어오도록
- **Warm tone**: 기존 베이지 배경의 따뜻한 톤 유지
- **Accessible**: 충분한 색상 대비, 터치 타겟 44px+

## 4. 구현 계획

### 4.1 작업 순서
1. CSS 스타일 수정 (인라인 `<style>` 블록)

3. 예산 소진 분석 영역 비주얼 업그레이드
4. 성과지표 카드 스타일 개선
5. 일정/게시글 목록 스타일 변경
6. 갤러리 썸네일 영역 개선
7. 반응형 확인 및 미세 조정

### 4.2 영향 범위
- `index.html` — `<style>` 블록 (CSS) + `renderDashboard()` 함수 (JSX)
- 다른 페이지에 영향 없음 (대시보드 전용 스타일 추가)

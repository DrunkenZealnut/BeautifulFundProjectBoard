# Plan: 학교관리페이지

## Executive Summary

| 항목 | 내용 |
|------|------|
| **Feature** | 학교관리페이지 |
| **시작일** | 2026-03-22 |
| **예상 기간** | 2~3일 |

| 관점 | 설명 |
|------|------|
| **Problem** | 반도체 특성화고 5개교(인천·충남·한국·대구 등)의 연락처, 교과서 현황, 전공 정보, 학생 수, 공지사항을 개별적으로 관리하고 있어 통합 현황 파악이 어려움 |
| **Solution** | 학교별 프로필 카드 + 교과서 목록 + 공지사항 실시간 모니터링을 하나의 관리 페이지로 통합 |
| **Function UX Effect** | 학교 관련 정보를 한 화면에서 즉시 확인·수정 가능, 공지사항 변경 시 알림 수신 |
| **Core Value** | 캠페인/위원회 운영 시 학교별 맞춤 대응이 가능하여 사업 효율성 향상 |

## 1. 배경 및 목적

### 1.1 사업 맥락
- 청년노동자인권센터는 반도체 특성화고 대상 순회 캠페인, 청소년참견위원회, 교과서 기초연구 사업 수행
- 현재 5개 학교와 협력 중: 인천반도체고, 충남반도체고, 한국반도체고, 대구반도체고, + 추가 예정
- 학교별 담당교사 연락처, 전공 구성, 학생 규모 등이 체계적으로 관리되지 않음

### 1.2 현재 문제
- 학교 정보가 개인 연락처/메모에 산재
- 캠페인 방문 시마다 학교 정보를 재확인해야 함
- 교과서 보급 현황 추적 불가
- 학교 공지사항(채용/안전 관련)을 수동으로 확인

### 1.3 목표
- 학교별 프로필(연락처, 전공, 학생 수) 통합 관리
- 교과서 배포/활용 목록 추적
- 학교 공지사항 URL 등록 + 실시간 모니터링(선택적)
- 캠페인/위원회 활동과 연계

## 2. 기능 요구사항

### 2.1 학교 프로필 관리 (CRUD)

| 필드 | 타입 | 설명 |
|------|------|------|
| `name` | text | 학교명 (예: 인천반도체고등학교) |
| `short_name` | text | 약칭 (예: 인천반도체고) |
| `address` | text | 주소 |
| `phone` | text | 대표 전화번호 |
| `website` | text | 학교 홈페이지 URL |
| `principal` | text | 교장 이름 |
| `contact_teacher` | text | 담당교사 이름 |
| `contact_teacher_phone` | text | 담당교사 연락처 |
| `contact_teacher_email` | text | 담당교사 이메일 |
| `departments` | text[] | 전공 목록 (예: ['반도체설계', '반도체공정', '반도체장비']) |
| `student_count` | integer | 전체 학생 수 |
| `student_count_by_grade` | jsonb | 학년별 학생 수 (예: {"1학년": 120, "2학년": 110, "3학년": 100}) |
| `notice_url` | text | 공지사항 페이지 URL (크롤링 대상) |
| `memo` | text | 비고/메모 |
| `status` | text | 상태 (active/inactive) |
| `partnership_date` | date | 협력 시작일 |

### 2.2 교과서 목록 관리

| 필드 | 타입 | 설명 |
|------|------|------|
| `school_id` | uuid | 학교 FK |
| `textbook_name` | text | 교과서명 |
| `subject` | text | 과목 (예: 반도체공정실습) |
| `publisher` | text | 출판사 |
| `year` | integer | 발행연도 |
| `is_safety_related` | boolean | 안전 관련 교과서 여부 |
| `our_textbook` | boolean | 센터 개발 교과서 여부 |
| `distribution_status` | text | 배포 상태 (planned/distributed/in_use) |
| `quantity` | integer | 배포 수량 |

### 2.3 공지사항 모니터링

| 기능 | 설명 |
|------|------|
| URL 등록 | 학교별 공지사항 페이지 URL 저장 |
| 수동 확인 | 버튼 클릭 시 새 탭에서 공지 페이지 열기 |
| 키워드 알림 (향후) | 안전/채용/실습 등 키워드 포함 공지 자동 감지 |

> **참고**: 실시간 자동 크롤링은 법적/기술적 제약이 있으므로, 1차 구현에서는 URL 링크 + 수동 확인 방식으로 진행. 향후 RSS/API 연동 가능 시 자동화 검토.

### 2.4 학교-활동 연계

- 일정(schedules)에 학교 태그 연결
- 갤러리(galleries)에 학교별 필터
- 캠페인 집행 내역과 학교 매핑

## 4. 기술 구현

### 4.1 DB 테이블 (Supabase)

```sql
-- 학교 테이블
CREATE TABLE IF NOT EXISTS public.schools (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name TEXT NOT NULL,
    short_name TEXT,
    address TEXT,
    phone TEXT,
    website TEXT,
    principal TEXT,
    contact_teacher TEXT,
    contact_teacher_phone TEXT,
    contact_teacher_email TEXT,
    departments TEXT[] DEFAULT '{}',
    student_count INTEGER DEFAULT 0,
    student_count_by_grade JSONB DEFAULT '{}',
    notice_url TEXT,
    memo TEXT,
    status TEXT DEFAULT 'active' CHECK (status IN ('active', 'inactive')),
    partnership_date DATE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 교과서 테이블
CREATE TABLE IF NOT EXISTS public.school_textbooks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    school_id UUID REFERENCES public.schools(id) ON DELETE CASCADE,
    textbook_name TEXT NOT NULL,
    subject TEXT,
    publisher TEXT,
    year INTEGER,
    is_safety_related BOOLEAN DEFAULT false,
    our_textbook BOOLEAN DEFAULT false,
    distribution_status TEXT DEFAULT 'planned' CHECK (distribution_status IN ('planned', 'distributed', 'in_use')),
    quantity INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- RLS 정책
ALTER TABLE public.schools ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.school_textbooks ENABLE ROW LEVEL SECURITY;

CREATE POLICY "schools_read" ON public.schools FOR SELECT USING (true);
CREATE POLICY "schools_write" ON public.schools FOR ALL USING (true);
CREATE POLICY "textbooks_read" ON public.school_textbooks FOR SELECT USING (true);
CREATE POLICY "textbooks_write" ON public.school_textbooks FOR ALL USING (true);
```

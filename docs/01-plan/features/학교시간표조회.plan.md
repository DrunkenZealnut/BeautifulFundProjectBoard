# Plan: 학교정보 — 학년별 전공별 시간표 가져오기

## Executive Summary

| 항목 | 내용 |
|------|------|
| **Feature** | 학교시간표조회 |
| **시작일** | 2026-03-23 |
| **예상 기간** | 1~2일 |

| 관점 | 설명 |
|------|------|
| **Problem** | 캠페인/교과서 사업 시 학교별 수업 시간표를 모르면 방문 시간 조율이 어렵고, 어떤 과목(특히 안전 관련)이 편성되어 있는지 파악 불가 |
| **Solution** | NEIS Open API `hisTimetable`을 통해 학년별/전공별 주간 시간표를 실시간으로 조회하여 학교 상세 페이지에 표시 |
| **Function UX Effect** | 학교 상세에서 날짜/학년/반 선택 → 시간표 즉시 확인, 안전 관련 과목 하이라이트 |
| **Core Value** | 캠페인 방문 시간 최적화 + 안전교육 교과 편성 현황 파악으로 사업 전략 수립 지원 |

## 1. 배경 및 목적

### 1.1 NEIS 시간표 API 검증 결과

API 엔드포인트: `https://open.neis.go.kr/hub/hisTimetable`

실제 테스트 (인천전자마이스터고, 2026-03-16):

| 필드 | 값 | 설명 |
|------|-----|------|
| `ATPT_OFCDC_SC_CODE` | E10 | 교육청코드 |
| `SD_SCHUL_CODE` | 7310370 | 학교코드 |
| `GRADE` | 1, 2, 3 | 학년 |
| `CLASS_NM` | 1, 2 | 반 |
| `DDDEP_NM` | 인공지능전자과 | 학과명 |
| `PERIO` | 1~7 | 교시 |
| `ITRT_CNTNT` | 디지털 논리 회로 등 | 과목명 |
| `ALL_TI_YMD` | 20260316 | 날짜 |
| `SEM` | 1 | 학기 |

확인된 반 구성: 1-1, 1-2, 2-1, 2-2, 3-1, 3-2 (학년당 2반)

주요 과목 (안전 관련 가능성):
- 하드웨어 분석, 하드웨어 성능 구현
- 디지털 논리 회로, 제품 회로설계
- 인공지능 플랫폼 인프라 구현
- 성공적인 직업생활 (직업안전 포함 가능)

### 1.2 목표
- 학교 상세 페이지에서 NEIS 시간표를 실시간 조회
- 학년/반/날짜 필터링
- 안전 관련 과목 하이라이트
- 주간 시간표 테이블 뷰

## 2. 기능 요구사항

### 2.2 필요한 필터

| 필터 | 옵션 | 기본값 |
|------|------|--------|
| 날짜 범위 | 이번 주 월~금 | 이번 주 |
| 학년 | 전체, 1학년, 2학년, 3학년 | 전체 |
| 반 | 전체, 1반, 2반, ... | 전체 |

### 2.3 안전 관련 과목 하이라이트

키워드 매칭으로 안전 관련 과목 자동 감지:
- "안전", "보건", "직업생활", "산업", "화학", "하드웨어", "실습"
- 해당 셀에 배경색 강조 + 범례 표시

## 3. 기술 구현

### 3.1 API 호출 (서버 프록시)

기존 `/api/neis.py` 프록시를 확장하여 시간표 조회 기능 추가:

```text
GET /api/neis?action=timetable&office=E10&school=7310370&from=20260316&to=20260320
GET /api/neis?action=timetable&office=E10&school=7310370&from=20260316&to=20260320&grade=1&class=1
```

### 3.2 NEIS API 필드 매핑

| NEIS 필드 | 용도 |
|-----------|------|
| `GRADE` | 학년 필터 |
| `CLASS_NM` | 반 필터 |
| `DDDEP_NM` | 학과(전공)명 표시 |
| `PERIO` | 교시 (행) |
| `ITRT_CNTNT` | 과목명 (셀 내용) |
| `ALL_TI_YMD` | 날짜 (열) |

### 3.3 데이터 구조 (클라이언트 상태)

### 3.4 주간 날짜 계산

```javascript
// 이번 주 월~금 날짜 계산
const getWeekDates = (baseDate) => {
    const d = new Date(baseDate);
    const day = d.getDay();
    const monday = new Date(d);
    monday.setDate(d.getDate() - (day === 0 ? 6 : day - 1));
    return Array.from({length: 5}, (_, i) => {
        const date = new Date(monday);
        date.setDate(monday.getDate() + i);
        return date.toISOString().split('T')[0].replace(/-/g, '');
    });
};
```

## 7. 제약 조건

- NEIS API 키는 서버 프록시(`/api/neis.py`) 경유 (클라이언트 노출 금지)
- schools 테이블의 `neis_code` + `neis_office_code` 필수 (없으면 시간표 조회 불가)
- NEIS API 응답 최대 1000건/요청 → 주간 단위 조회로 제한
- 시간표 데이터는 DB에 저장하지 않음 (실시간 조회만)

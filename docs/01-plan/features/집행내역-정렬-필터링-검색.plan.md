## 1. 현황 분석

### 1.1 현재 상태
- **위치**: `index.html` 예산관리 페이지 → `budgetTab === 'history'` (line ~7886)
- **데이터 소스**: `data.budgetExecutions` (Supabase `budget_executions` 테이블, `created_at DESC` 정렬)
- **렌더링**: `data.budgetExecutions.map(execution => ...)` — 전체 목록 단순 반복
- **기존 기능**: 수정, 삭제, 승인 상태 변경, Excel 내보내기, 급여명세서/회의일지 생성

### 1.2 문제점
1. **정렬 불가**: 날짜, 금액, 카테고리 등 기준으로 정렬할 수 없음
2. **필터링 불가**: 특정 카테고리, 상태, 결제방법, 기간별 조회 불가
3. **검색 불가**: 설명, 수취인, 항목명 등 키워드 검색 불가
4. **데이터 증가 시 UX 저하**: 모든 항목이 한 번에 표시되어 스크롤이 길어짐

### 1.3 데이터 필드 (budget_executions)
| 필드 | 용도 | 정렬/필터 대상 |
|------|------|----------------|
| `execution_date` | 집행일 | 정렬, 기간 필터 |
| `amount` | 금액 | 정렬 |
| `category_name` | 대분류 (사업비/운영비) | 필터 |
| `subcategory_name` | 소분류 | 필터 |
| `budget_item_name` | 예산 항목명 | 검색 |
| `description` | 설명 | 검색 |
| `recipient` | 수취인 | 검색 |
| `payment_method` | 결제방법 (카드/계좌이체) | 필터 |
| `status` | 상태 (pending/approved/executed/completed) | 필터 |
| `type` | 비용 유형 | 필터 |
| `created_at` | 등록일 | 정렬 |

## 2. 요구사항

### 2.1 정렬 기능
| 정렬 기준 | 방향 | 기본값 |
|-----------|------|--------|
| 집행일 (`execution_date`) | 최신순 / 오래된순 | 최신순 (기본) |
| 금액 (`amount`) | 높은순 / 낮은순 | - |
| 등록일 (`created_at`) | 최신순 / 오래된순 | - |

### 2.2 필터링 기능
| 필터 | 타입 | 옵션 |
|------|------|------|
| 카테고리 | select | 전체 / 사업비 / 운영비 |
| 소분류 | select (카테고리 연동) | 전체 / 해당 카테고리의 소분류 목록 |
| 상태 | select | 전체 / 대기 / 승인 / 집행완료 / 완료 |
| 결제방법 | select | 전체 / 카드 / 계좌이체 |
| 기간 | date range (시작~종료) | - |

### 2.3 검색 기능
| 항목 | 설명 |
|------|------|
| 검색 대상 | `budget_item_name`, `description`, `recipient` |
| 검색 방식 | 클라이언트 사이드, 부분 일치 (includes), 대소문자 무시 |
| UI | 텍스트 입력 + 실시간 필터링 (debounce 불필요 — 데이터가 이미 메모리에 있음) |

### 2.4 결과 표시
| 항목 | 설명 |
|------|------|
| 건수 표시 | 필터링된 결과 수 / 전체 수 표시 (예: "12건 / 45건") |
| 빈 결과 | "조건에 맞는 집행 내역이 없습니다." 메시지 |
| 필터 초기화 | "초기화" 버튼으로 모든 필터/검색/정렬을 기본값으로 리셋 |

### 3.2 필터링 + 정렬 로직 (useMemo)
- `data.budgetExecutions`를 입력으로 받아 필터 → 검색 → 정렬 파이프라인 적용
- `useMemo`로 메모이제이션하여 불필요한 재계산 방지

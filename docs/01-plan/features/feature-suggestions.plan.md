# Feature Suggestions Plan - 현재 프로젝트에 추가하면 좋을 기능 제안

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 현재 시스템은 핵심 CRUD와 예산 집행 관리가 잘 구현되어 있으나, 일상 운영에서 필요한 알림/리마인더, 활동 추적, 데이터 분석 깊이, 사용자 편의 기능이 부족하여 사용자가 수동으로 확인해야 할 사항이 많음 |
| **Solution** | 사업 운영 효율성과 데이터 활용도를 높이는 10개 기능을 3단계 우선순위로 제안 — 즉시 적용 가능한 UX 개선부터 중장기 분석 기능까지 |
| **Function UX Effect** | 알림으로 누락 방지, 댓글로 소통 활성화, PDF 직접 생성으로 행정 효율 향상, 대시보드 강화로 의사결정 지원 |
| **Core Value** | 공익단체의 제한된 인력으로도 7천만원 예산을 투명하고 빈틈없이 관리할 수 있는 자동화된 운영 도구 |

---

## 1. 현재 시스템 현황 분석

### 1.1 구현 완료된 핵심 기능 (150+)

| 모듈 | 주요 기능 | 완성도 |
|------|----------|:------:|
| 대시보드 | 집행률, 성과지표, 월별 차트, 최근 활동 | 90% |
| 예산관리 | 편성표, 집행등록, 원천징수, 집행내역, 캘린더, 비용비율, 사업변경신청 | 95% |
| 일정관리 | 캘린더, 목록, Google Calendar 연동, CRUD | 90% |
| 갤러리 | 그리드/상세 뷰, Rich Text, 파일 업로드, 카테고리 필터 | 85% |
| 게시판 | CRUD, Rich Text, 카테고리, 페이지네이션, 파일 첨부 | 85% |
| 회계가이드 | 기본원칙, 계정항목, 적격증빙, FAQ, 원천징수 계산기 | 90% |
| 관리자 | 사용자/수급자/기관정보 관리 | 90% |
| 인증 | 로그인/로그아웃, 역할 기반 접근 제어, RLS | 85% |

### 1.2 식별된 개선 영역

현재 시스템에서 **가장 큰 가치를 추가할 수 있는 영역**을 다음 기준으로 평가:

- **사용 빈도**: 매일 사용하는 기능인가?
- **운영 효율**: 수동 작업을 줄여주는가?
- **사업 보고**: 아름다운재단 보고 요구에 도움이 되는가?
- **구현 난이도**: 현재 아키텍처(단일 파일 SPA)에서 실현 가능한가?

## 2. 기능 제안 (우선순위별)

### Priority 1: 즉시 적용 가능 (1-2일 소요)

#### F-01: 마감일 알림 & 리마인더 배너

| 항목 | 내용 |
|------|------|
| **문제** | 예산 집행 마감, 보고서 제출 마감, 일정 등을 수동으로 확인해야 함 |
| **해결** | 대시보드 상단에 D-day 기반 알림 배너 표시 |
| **구현 범위** | 대시보드 페이지 상단 |
| **난이도** | 낮음 |

**세부 기능:**
- 7일 이내 다가오는 일정 알림 (D-7, D-3, D-1, D-Day)
- 반기별 정산 마감 알림 (6월 30일, 12월 31일)
- 미승인 집행건 알림 (pending 상태 N건)
- 색상 구분: 빨강(긴급 D-1~D-Day), 주황(주의 D-3), 파랑(정보 D-7)
- 닫기(dismiss) 기능

```
예시 UI:
┌─────────────────────────────────────────────────┐
│ 🔴 [D-1] 운영위원회 회의 내일 예정 (3/11)        │
│ 🟠 [D-3] 미승인 집행건 5건 처리 필요              │
│ 🔵 [D-7] 1차 반기 정산 마감 (6/30)               │
└─────────────────────────────────────────────────┘
```

#### F-02: 게시판/갤러리 댓글 기능

| 항목 | 내용 |
|------|------|
| **문제** | 게시글이나 갤러리에 대한 피드백/논의를 할 수 없음 |
| **해결** | 게시글, 갤러리 상세 페이지에 댓글 기능 추가 |
| **구현 범위** | boards, galleries 상세 뷰 하단 |
| **난이도** | 중간 |

**세부 기능:**
- 댓글 작성/수정/삭제
- 작성자명, 작성 시간 표시
- 본인 댓글만 수정/삭제 가능 (admin은 전체 삭제 가능)
- Supabase `comments` 테이블 신규 필요:
  ```sql
  comments (id UUID, parent_type TEXT, parent_id UUID,
            content TEXT, author_id UUID, created_at TIMESTAMPTZ)
  ```

#### F-03: 사용자 프로필 & 비밀번호 변경

| 항목 | 내용 |
|------|------|
| **문제** | 사용자가 자신의 비밀번호를 변경할 수 없음 (관리자만 가능) |
| **해결** | 헤더 사용자명 클릭 → 프로필 모달 (이름, 이메일, 비밀번호 변경) |

- 현재 비밀번호 확인 후 새 비밀번호 설정
- 이름, 이메일, 전화번호 자기 수정
- 비밀번호 강도 표시 (약함/보통/강함)

### Priority 2: 사업 운영 강화 (3-5일 소요)

#### F-04: 활동 로그 (Audit Trail)

| 항목 | 내용 |
|------|------|
| **문제** | 누가 언제 무엇을 변경했는지 추적할 수 없음 |
| **해결** | 주요 작업(집행 등록/수정/삭제, 상태 변경, 게시글 작성 등)에 대한 활동 로그 기록 |
| **구현 범위** | 관리자 페이지 새 탭 + 각 모듈의 CUD 작업 후크 |
| **난이도** | 중간 |

**세부 기능:**
- Supabase `activity_logs` 테이블:
  ```sql
  activity_logs (id UUID, user_id UUID, action TEXT,
                 target_type TEXT, target_id UUID,
                 details JSONB, created_at TIMESTAMPTZ)
  ```
- 관리자 페이지에 '📜 활동 로그' 탭 추가
- 필터: 사용자별, 작업 유형별, 기간별
- 최근 활동 대시보드 위젯

```
예시:
[2026-03-10 14:32] 김철수 - 집행 등록 "캠페인 인쇄비" 350,000원
[2026-03-10 14:15] 박영희 - 게시글 작성 "3월 활동 보고서"
[2026-03-10 13:50] admin  - 집행 상태 변경 "강사료" pending→approved
```

#### F-05: 대시보드 기간별 비교 분석

| 항목 | 내용 |
|------|------|
| **문제** | 월별 집행 추이 차트만 있고, 계획 대비 실적 비교가 어려움 |
| **해결** | 대시보드에 '예산 소진 속도' 분석 위젯 추가 |
| **구현 범위** | 대시보드 페이지 |
| **난이도** | 중간 |

**세부 기능:**
- 예산 소진 속도 표시: 현재 집행률 vs 기간 경과율
  - "현재 기간 경과율 25% / 예산 집행률 30% → 약간 초과 집행 중"
- 월별 예상 집행액 vs 실제 집행액 라인 차트
- 잔여 예산으로 남은 기간 동안 월평균 사용 가능 금액 표시
- 카테고리별 과부족 알림 (특정 항목 예산 80% 이상 소진 시 경고)

```
예시 UI:
┌─────────────────────────────────────────────┐
│ 📈 예산 소진 분석                             │
│                                               │
│ 기간 경과: ████████░░░░░░░░ 25% (3/12개월)    │
│ 예산 집행: ██████████░░░░░░ 30%               │
│                                               │
│ ⚠️ 집행 속도가 기간 대비 1.2배 빠릅니다        │
│ 💡 남은 9개월간 월평균 5,444,444원 사용 가능   │
│                                               │
│ 🔴 캠페인비 예산 소진율 85% (주의)             │
│ 🟢 교과서기초연구 예산 소진율 15% (양호)        │
└─────────────────────────────────────────────┘
```

#### F-06: PDF 직접 생성 (정산보고서)

| 항목 | 내용 |
|------|------|
| **문제** | 현재 브라우저 인쇄(Ctrl+P)로만 PDF 저장 가능. 자동화된 정산보고서 생성 불가 |
| **해결** | 반기별/연간 정산보고서를 PDF로 직접 생성하는 기능 |
| **구현 범위** | 예산 관리 페이지 새 탭 또는 대시보드 |
| **난이도** | 높음 |

**세부 기능:**
- jsPDF + html2canvas CDN 라이브러리 활용
- 보고서 종류:
  1. **집행 총괄표**: 카테고리별 예산/집행/잔액 요약
  2. **월별 집행 명세서**: 상세 집행 내역 테이블
  3. **급여지급명세서**: 기존 인쇄 기능의 PDF 버전
- 아름다운재단 제출 양식에 맞는 레이아웃
- 기관 직인 이미지 자동 삽입

#### F-09: 오프라인 모드 (PWA)

| 항목 | 내용 |
|------|------|

- 온라인 복귀 시 자동 데이터 갱신

## 5. 기술 제약 사항

| 제약 | 영향 | 대응 |
|------|------|------|
| 단일 파일 SPA (~8000줄) | 파일 크기 계속 증가 | 기능별 코드 섹션 주석으로 구분 |
| CDN 기반 (no build) | 라이브러리 추가 시 CDN URL 필요 | unpkg/jsdelivr 활용 |
| Supabase Free Tier | 데이터베이스 용량/API 호출 제한 | 효율적 쿼리, 불필요 데이터 정리 |
| 브라우저 인쇄 의존 | PDF 품질 일관성 부족 | jsPDF CDN 도입으로 해결 가능 |

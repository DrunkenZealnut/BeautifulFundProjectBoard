## Executive Summary

| Perspective | Content |
|-------------|---------|
| **Problem** | 5종 템플릿이 각각 schedules/boards/gallery 렌더링을 독립 구현(779줄 중복), 섹션 순서가 하드코딩되어 사용자 제어 불가, Step 3에서 타입 내 이동만 가능 |
| **Solution** | 공통 섹션 렌더러 도입으로 중복 제거, 섹션 순서를 orderedSections 배열로 관리하여 사용자가 Step 3에서 자유 배치 |

## 1. 현재 구조 분석

### 1.1 중복 패턴

각 템플릿에서 반복되는 로직:
- `selectedSchedules.map(s => ...)` — 일정 렌더링 (5곳)
- `selectedBoards.map(b => ...)` + `boardImageUrls[b.id]` — 게시글 렌더링 (5곳)  
- `selectedGalleries.map(g => ...)` + `galleryThumbUrls[g.id]` — 갤러리 렌더링 (5곳)
- `config.greeting`, `config.closing` 처리 (5곳)
- `_nlBaseHTML()` 호출 (5곳)

### 1.2 섹션 순서 현황

| 템플릿 | 순서 |
|--------|------|
| Modern | greeting → schedules → boards → gallery → closing |
| Magazine | greeting → boards → schedules → gallery → closing |
| Classic | greeting → boards → schedules → gallery → closing |
| Email | greeting → boards → schedules → gallery → closing |
| Grid | greeting → all-mixed → closing |

## 2. 리팩토링 설계

### 2.1 공통 섹션 렌더러 아키텍처

```javascript
// 섹션 렌더러: 템플릿 스타일 설정을 받아 HTML 문자열 반환
const _nlRenderSection = (type, data, theme) => {
    // type: 'schedules' | 'boards' | 'gallery'
    // data: { items, boardImageUrls, rewrittenContents, galleryThumbUrls, esc }
    // theme: { sectionTitle, itemStyle, imageStyle, ... } — 템플릿별 스타일
};
```

### 2.2 테마 객체 구조

각 템플릿은 **스타일 테마 객체**만 정의하면 됨:

```javascript
const NL_THEMES = {
    modern: {
        name: '모던 / 미니멀',
        layout: 'flex',                    // 'flex' | 'table' | 'grid'
        accent: '#0d9488',
        font: "'Pretendard', -apple-system, sans-serif",
        sectionLabel: { fontSize: '11px', fontWeight: 700, letterSpacing: '1px' },
        header: { ... },
        footer: { ... },
        scheduleItem: { ... },
        boardItem: { ... },
        galleryGrid: { columns: 3, gap: '8px', height: '120px' },
        greeting: { ... },
        closing: { ... },
    },
    magazine: { ... },
    classic: { ... },
    email: { layout: 'table', ... },       // table-based
    grid: { layout: 'grid-cards', ... },   // card grid
};
```

### 2.3 리팩토링된 generate 함수

```javascript
const generateNewsletterHTML = (template, config, sections, orgName, assets) => {
    // sections: orderedSections 배열 [{ type: 'boards', items: [...] }, { type: 'schedules', items: [...] }, ...]
    // assets: { boardImageUrls, rewrittenContents, galleryThumbUrls }
    const theme = NL_THEMES[template] || NL_THEMES.modern;
    const esc = _nlEsc;
    
    // 각 섹션을 순서대로 렌더링
    const sectionsHtml = sections.map(sec => 
        _nlRenderSection(sec.type, sec.items, theme, assets, esc)
    ).join('');
    
    // 헤더 + 인사말 + 섹션들 + 마무리말 + 푸터 조합
    const body = _nlRenderLayout(theme, config, orgName, sectionsHtml, esc);
    const style = _nlGetStyle(theme);
    return _nlBaseHTML(config.title || '뉴스레터', style, body);
};
```

### 2.4 Grid 템플릿 특수 처리

Grid는 다른 템플릿과 달리 타입 무관하게 카드로 렌더링합니다:

```javascript
if (theme.layout === 'grid-cards') {
    // 모든 섹션의 아이템을 하나의 카드 그리드로 합침
    const allCards = sections.flatMap(sec => sec.items.map(item => ({ ...item, sectionType: sec.type })));
    return _nlRenderGridCards(allCards, theme, assets, esc);
}
```

## 3. Step 3 배치 UI 고도화

### 3.1 섹션 단위 + 아이템 단위 2-레벨 순서 변경

```
┌─────────────────────────────┐
│ 콘텐츠 배치                   │
│                              │
│ ┌──────────────────────┐ [↑↓]│  ← 섹션 단위 이동
│ │ 📋 주요 소식 (2건)     │    │
│ │  ├ 워크숍 성료     [↑↓]│    │  ← 아이템 단위 이동
│ │  └ 캠페인 시작     [↑↓]│    │
│ └──────────────────────┘     │
│ ┌──────────────────────┐ [↑↓]│
│ │ 📅 주요 일정 (3건)     │    │
│ │  ├ 04-10 워크숍    [↑↓]│    │
│ │  ├ 04-15 캠페인    [↑↓]│    │
│ │  └ 04-22 학교방문  [↑↓]│    │
│ └──────────────────────┘     │
│ ┌──────────────────────┐ [↑↓]│
│ │ 📸 갤러리 (1건)        │    │
│ │  └ 워크숍 현장     [↑↓]│    │
│ └──────────────────────┘     │
└─────────────────────────────┘
```

### 3.2 상태 구조 변경

```javascript
// 기존: orderedItems (플랫 배열)
// 변경: orderedSections (섹션 배열, 각 섹션 내 아이템 배열)
const [orderedSections, setOrderedSections] = useState([]);
// 형태: [
//   { type: 'boards', label: '주요 소식', items: [board1, board2] },
//   { type: 'schedules', label: '주요 일정', items: [sched1, sched2, sched3] },
//   { type: 'gallery', label: '갤러리', items: [gal1] },
// ]
```

### 3.3 이동 함수

```javascript
// 섹션 순서 이동
const moveSection = (index, direction) => {
    const ni = index + direction;
    if (ni < 0 || ni >= orderedSections.length) return;
    const a = [...orderedSections];
    [a[index], a[ni]] = [a[ni], a[index]];
    setOrderedSections(a);
};

// 섹션 내 아이템 순서 이동
const moveItemInSection = (sectionIndex, itemIndex, direction) => {
    const ni = itemIndex + direction;
    const sec = orderedSections[sectionIndex];
    if (ni < 0 || ni >= sec.items.length) return;
    const items = [...sec.items];
    [items[itemIndex], items[ni]] = [items[ni], items[itemIndex]];
    const sections = [...orderedSections];
    sections[sectionIndex] = { ...sec, items };
    setOrderedSections(sections);
};
```

## 4. 템플릿 디자인 개선

### 4.1 Modern (현재 78줄)
- **개선**: 여백 확대, 날짜 포맷 개선, 게시글 이미지 라운딩 강화
- 현재 충분히 깔끔 → 소폭 조정

### 4.2 Magazine (현재 111줄)
- **개선**: Featured 카드 그림자 강화, 카테고리 배지 컬러 통일, 카드 간 간격 조정

### 4.3 Classic (현재 103줄)
- **개선**: 표 헤더 컬러 조정, 더블 보더 → 싱글로 변경, 기관 로고 영역 추가

### 4.4 Email (현재 259줄)
- **개선**: 히어로 배너 배경색 다양화, 소셜 아이콘 영역 추가, 반응형 개선

### 4.5 Grid (현재 188줄)
- **개선**: Featured 카드 높이 자동 조절, 카드 호버 없이 그림자 기본 적용

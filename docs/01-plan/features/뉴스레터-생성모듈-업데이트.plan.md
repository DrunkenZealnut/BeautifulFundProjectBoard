## Executive Summary

| Perspective | Content |
|-------------|---------|
| **Problem** | 뉴스레터 작성 상태가 메모리에만 존재해 새로고침 시 전부 소실되고 과거 발행본 재활용 불가. 템플릿 색상·폰트가 하드코딩되어 단체 브랜딩 반영 불가. Step 3에서 텍스트 수정하려면 Step 1로 되돌아가야 함 |

| **Core Value** | 발행 주기마다 처음부터 다시 만들던 뉴스레터를 "지난 호 복제 → 콘텐츠 교체 → 발행" 3단계로 단축 (예상 제작 시간 50%+ 절감), 발행 이력 아카이브 확보 |

## 1. 현재 구조 분석

### 1.1 상태 관리 (index.html ~line 4342-4357)

| 상태 | 내용 | 휘발성 |
|------|------|--------|
| `newsletterConfig` | title, issueNumber, publishDate, greeting, closing, 필터(dateFrom/To, category) | 새로고침 시 소실 |
| `newsletterTemplate` | 'modern' \| 'magazine' \| 'classic' \| 'email' \| 'grid' | 〃 |
| `selectedScheduleIds` / `selectedBoardIds` / `selectedGalleryNewsletterIds` | 콘텐츠 선택 (Set) | 〃 |
| `orderedSections` | Step 3 배치 결과 `[{type, label, items}]` | 〃 |
| `itemLinks` | 아이템별 링크 `{[itemId]: url}` | 〃 |
| `rewrittenContents` | AI 재작성 본문 `{[boardId]: text}` | 〃 |
| `boardImageUrls` / `galleryThumbUrls` | 이미지 URL 맵 | 〃 |

### 1.2 출력 수단 (Step 3)

- `handlePrintOrdered()` — 팝업 인쇄
- `handleCopyOrdered()` — 클립보드 복사 (text/html + text/plain)
- **HTML 파일 다운로드 없음** — 이메일 발송 도구·아카이빙에 쓸 원본 파일 확보 불가

### 1.3 템플릿 시스템

- Modern/Magazine/Classic: `NL_THEMES` 테마 객체 (accent, accent2, bg, font 하드코딩) → `_nlGenerateThemed()` 공통 렌더러
- Email/Grid: 독립 generate 함수 (색상 인라인 하드코딩)
- 색상·폰트를 바꾸려면 코드 수정 필요

### 1.4 DB 현황

`bf` 스키마에 뉴스레터 관련 테이블 없음 (projects.modules 배열에 "newsletter" 모듈 플래그만 존재)

## 2. 요구사항

### F-1. 저장·불러오기 (우선순위: 높음)

| ID | 요구사항 | 세부 |
|----|----------|------|
| F-1-1 | `bf.newsletters` 테이블 신설 | id(UUID), project_id, title, issue_number, template, config(JSONB), sections(JSONB), assets(JSONB), status('draft'\|'published'), created_at, updated_at + RLS |
| F-1-2 | 임시저장 | Step 2·3에서 "임시저장" 버튼 → 현재 상태 전체 직렬화 저장 (같은 draft 재저장 시 UPDATE) |
| F-1-3 | 저장 목록 | 뉴스레터 페이지 진입 화면(Step 1 상단 또는 별도 영역)에 저장본 목록 — 제목·호수·상태·수정일 표시 |
| F-1-4 | 불러오기 | 목록에서 선택 → 전체 상태 복원 후 Step 3 (또는 저장 시점 Step)으로 이동 |
| F-1-5 | 복제 | 지난 호 복제 → 새 draft 생성 (호수·발행일 초기화) |
| F-1-6 | 삭제 | 저장본 삭제 (확인 다이얼로그) |
| F-1-7 | 발행 처리 | 인쇄/복사/다운로드 실행 시 status='published' 전환 제안 (아카이브 목적) |

**설계 쟁점 (Design 단계에서 확정)**:
- **스냅샷 vs ID 참조**: `orderedSections`의 items를 통째로 저장(스냅샷)하면 원본 게시글이 수정·삭제돼도 발행본 보존 가능. ID만 저장하면 용량은 작지만 재현 불가. → **스냅샷 방식 권장** (발행 이력 보존이 목적이므로)
- 이미지 URL 만료 여부 확인 필요 (Supabase Storage public URL이면 문제없음)

### F-2. 템플릿·디자인 확장 (우선순위: 중간)

| ID | 요구사항 | 세부 |
|----|----------|------|
| F-2-1 | 색상 커스터마이징 | Step 2에서 accent 색상 피커 (기본값 = 템플릿 고유색) → `NL_THEMES` accent/accent2 오버라이드 |
| F-2-2 | 폰트 선택 | 2~3종 폰트 스택 (Pretendard 고딕 / 명조 계열 / 시스템 기본) |
| F-2-3 | Email/Grid 커스터마이징 적용 | 독립 함수 2종도 색상 파라미터 수용하도록 리팩토링 (config.customColors 전달) |
| F-2-4 | 신규 템플릿 1종 | `NL_THEMES` 아키텍처로 추가 (후보: 포스터형/단신형 — Design 단계에서 확정) |

### F-3. 편집 UX 개선 (우선순위: 중간)

| ID | 요구사항 | 세부 |
|----|----------|------|

| F-3-2 | 섹션 라벨 편집 | "주요 소식" 등 섹션 제목을 사용자 정의로 변경 |

## 3. 범위

### In Scope
- `bf.newsletters` 테이블 + `supabase-migration-newsletters.sql` 마이그레이션
- index.html 내 뉴스레터 모듈 수정 (저장/불러오기 UI, 커스터마이징 UI, 인라인 편집)
- `NL_THEMES` 색상/폰트 오버라이드 메커니즘

### Out of Scope (명시적 제외)
- 이메일 실제 발송 기능 (SMTP/발송 API) — 별도 피처로 분리
- 공유 링크 (공개 URL 호스팅) — 별도 피처로 분리
- 드래그 앤 드롭 배치 (기존 ↑↓ 버튼 유지)
- WYSIWYG 리치 에디터 (인라인 편집은 plain text)

---

## 4. 제약 조건

- **단일 파일 SPA**: 모든 코드는 index.html 내 인라인. Babel standalone 제약 — `render*()` 헬퍼 내부 훅 사용 금지 (최상위에서만)
- **bf 스키마 + RLS**: 기존 테이블 패턴 (UUID PK, created_at/updated_at) 준수
- **직렬화 크기**: sections 스냅샷 JSONB가 커질 수 있음 (게시글 본문 포함) — 이미지 자체는 URL만 저장, base64 금지
- **단일 사용자**: 동시 편집 충돌 고려 불필요

## 7. 성공 기준

1. 작성 중 뉴스레터를 임시저장 → 브라우저 재시작 → 불러오기 시 Step 3 상태(배치·링크·재작성 본문 포함) 100% 복원
2. 지난 호 복제 → 호수만 변경 → 발행까지 5분 내 가능
3. 색상 피커로 지정한 accent가 5종 템플릿 모두에 반영
4. 커스터마이징 미사용 시 기존 5종 템플릿 출력 결과 변화 없음 (회귀 없음)
5. Step 3에서 제목·본문 수정이 미리보기에 즉시 반영, 원본 게시글은 불변

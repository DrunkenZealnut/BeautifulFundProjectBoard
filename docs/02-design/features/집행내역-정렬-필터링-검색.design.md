## 3. 데이터 파이프라인 설계

### 3.1 `filteredExecutions` (useMemo)

```javascript
const filteredExecutions = useMemo(() => {
    let result = data?.budgetExecutions || [];

    // 1) 카테고리 필터
    if (execFilterCategory) {
        result = result.filter(e => e.category_name === execFilterCategory);
    }

    // 2) 소분류 필터
    if (execFilterSubcategory) {
        result = result.filter(e => e.subcategory_name === execFilterSubcategory);
    }

    // 3) 상태 필터
    if (execFilterStatus) {
        result = result.filter(e => e.status === execFilterStatus);
    }

    // 4) 결제방법 필터
    if (execFilterPayment) {
        result = result.filter(e => e.payment_method === execFilterPayment);
    }

    // 5) 기간 필터
    if (execFilterDateFrom) {
        result = result.filter(e => e.execution_date >= execFilterDateFrom);
    }
    if (execFilterDateTo) {
        result = result.filter(e => e.execution_date <= execFilterDateTo);
    }

    // 6) 키워드 검색 (항목명, 설명, 수취인)
    if (execSearchQuery.trim()) {
        const q = execSearchQuery.trim().toLowerCase();
        result = result.filter(e =>
            (e.budget_item_name || '').toLowerCase().includes(q) ||
            (e.description || '').toLowerCase().includes(q) ||
            (e.recipient || '').toLowerCase().includes(q)
        );
    }

    // 7) 정렬
    result = [...result].sort((a, b) => {
        let va = a[execSortBy];
        let vb = b[execSortBy];
        // 숫자 필드
        if (execSortBy === 'amount') {
            va = va || 0;
            vb = vb || 0;
            return execSortDir === 'asc' ? va - vb : vb - va;
        }
        // 문자열/날짜 필드
        va = va || '';
        vb = vb || '';
        const cmp = va.localeCompare(vb);
        return execSortDir === 'asc' ? cmp : -cmp;
    });

    return result;
}, [
    data?.budgetExecutions,
    execFilterCategory, execFilterSubcategory, execFilterStatus,
    execFilterPayment, execFilterDateFrom, execFilterDateTo,
    execSearchQuery, execSortBy, execSortDir
]);
```

**삽입 위치**: `getAllBudgetItems` 함수 아래 (~line 3761)

### 3.2 카테고리-소분류 연동 로직
- 카테고리 필터 변경 시 → `setExecFilterSubcategory('')` 초기화
- 소분류 옵션: `BUDGET_DATA.categories`에서 선택된 카테고리의 subcategories만 표시

### 3.3 초기화 함수

```javascript
const resetExecFilters = () => {
    setExecSortBy('execution_date');
    setExecSortDir('desc');
    setExecFilterCategory('');
    setExecFilterSubcategory('');
    setExecFilterStatus('');
    setExecFilterPayment('');
    setExecFilterDateFrom('');
    setExecFilterDateTo('');
    setExecSearchQuery('');
};
```

    <button className="btn btn-secondary"
        style={{ fontSize: '12px', padding: '6px 10px' }}

## 7. 영향 범위 분석

### 7.1 영향 없는 기능 (확인 필요)
| 기능 | 이유 |
|------|------|
| 집행등록 (`budgetTab === 'register'`) | 별도 탭, `data.budgetExecutions`에 직접 push 후 상태 갱신 |
| 수정/삭제/승인 | `execution.id`로 동작 — 필터링 여부와 무관 |
| 급여지급명세서 | `execution` 객체를 직접 전달 — 필터링 무관 |
| 회의일지 | `execution` 객체를 직접 전달 — 필터링 무관 |
| 증빙서류 | `execution.id`로 매핑 — 필터링 무관 |
| 대시보드 탭 | `data.budgetExecutions`를 직접 사용 — 영향 없음 |
| 비용비율 탭 | `data.budgetExecutions`를 직접 사용 — 영향 없음 |

### 7.2 유일한 변경 영향
- Excel 내보내기: 필터링된 결과만 출력 (의도된 동작)
- 집행내역 탭의 목록 렌더링: `filteredExecutions` 사용 (의도된 동작)

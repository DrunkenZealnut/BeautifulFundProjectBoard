## 1. Architecture Overview

### 1.1 현재 구조 (Before)

```
renderNewsletter()                          ← 584줄 단일 함수
├── 템플릿 선택 카드 (5종)                    ← 상단
├── 2-column grid                           ← 중앙
│   ├── 좌측: 뉴스레터 정보 + 기간필터 + 일정/게시글/갤러리 선택
│   └── 우측: 인라인 미리보기 + 인쇄/복사/한글 버튼
└── NL_STYLES 기반 미리보기 렌더링              ← 하단
```

### 1.2 리뉴얼 구조 (After)

```
renderNewsletter()                          ← 위저드 라우터 (~30줄)
├── renderNLStepper()                       ← 스테퍼 UI (~30줄)
├── newsletterStep === 1
│   └── renderNLStep1()                     ← 내용 선택 (~200줄, 기존 코드 재활용)
├── newsletterStep === 2
│   └── renderNLStep2()                     ← 템플릿 선택 (~80줄)
└── newsletterStep === 3
    └── renderNLStep3()                     ← 배치/편집 + 미리보기 (~250줄, 신규)
```

**총 코드량**: 기존 584줄 → 리팩토링 후 ~590줄 (증가 최소화)

#### 3.4.1 orderedItems 동기화 로직

```javascript
// Step 3 진입 시 (newsletterStep이 3으로 변경될 때) orderedItems 동기화
useEffect(() => {

    const items = [];
    // 게시글 먼저, 일정, 갤러리 순서로 초기 배치
    (data.boards || []).filter(b => selectedBoardIds.has(b.id))

    (data.schedules || []).filter(s => selectedScheduleIds.has(s.id))
        .forEach(s => items.push({ type: 'schedule', id: s.id, data: s }));
    (data.galleries || []).filter(g => selectedGalleryNewsletterIds.has(g.id))
        .forEach(g => items.push({ type: 'gallery', id: g.id, data: g }));
    setOrderedItems(items);
}, [newsletterStep]);
```

#### 3.4.2 순서 변경 함수

```javascript
const moveItem = (index, direction) => {
    // direction: -1 (up) or +1 (down)
    const newIndex = index + direction;
    if (newIndex < 0 || newIndex >= orderedItems.length) return;
    setOrderedItems(prev => {
        const arr = [...prev];
        [arr[index], arr[newIndex]] = [arr[newIndex], arr[index]];
        return arr;
    });
};
```

## 7. 기존 미리보기 (NL_STYLES) 처리

현재 renderNewsletter 우측에 있는 인라인 미리보기 코드 (~12142–12412, 약 270줄)는:

- **Step 3의 iframe 미리보기로 완전 대체**
- NL_STYLES 객체 및 관련 렌더링 코드 **삭제**
- iframe `srcDoc={previewHtml}`이 실제 generateNewsletterHTML 출력을 표시하므로 더 정확한 미리보기 제공

**삭제 대상 코드**: line 12142 ~ 12412 (NL_STYLES 기반 인라인 미리보기)
**절감 효과**: ~270줄 삭제 → 신규 Step3 ~250줄 추가 = 순증 -20줄

## 9. Risk & Edge Cases

| Case | 대응 |
|------|------|
| Step 1에서 선택 후 Step 3까지 갔다가 Step 1로 돌아와서 선택 변경 | orderedItems는 Step 3 진입 시 재동기화 (useEffect) |

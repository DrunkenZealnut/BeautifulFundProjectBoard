## 1. Architecture Overview

### 1.1 현재 구조 (Before)

```
[Step 1 내용선택] → [Step 2 템플릿] → [Step 3 배치/편집] → 인쇄/복사
     │                    │                  │
     └── 모든 상태 메모리 전용 (새로고침 시 소실) ──┘

generateNewsletterHTML(template, config, sections, orgName, assets)
  ├─ themed (modern/magazine/classic) → _nlGenerateThemed — 색상 하드코딩
  ├─ email → generateNewsletterEmail — accentColor 로컬 상수
  └─ grid  → generateNewsletterGrid  — 색상 인라인
```

**확인된 제약** (설계 근거):
- 테마 렌더 함수는 `theme.accent`를 참조하지 않음 (grep 결과 0건) — hex가 HTML 문자열에 직접 하드코딩
- `_nlRenderSections`가 `sec.label`을 무시하고 '주요 소식'/'주요 일정'/'갤러리'를 하드코딩
- `goToStep(3)`이 진입 시마다 `buildSections()`로 orderedSections를 재생성 → Step 2 왕복 시 사용자 배치 소실 (잠재 결함)
- 이미지는 전부 `getPublicUrl` (public URL) — 만료 없음, 원본 삭제 시에만 깨짐

### 1.2 업데이트 구조 (After)

```
[저장된 뉴스레터 목록] ⇄ bf.newsletters (JSONB 스냅샷)
     │ 불러오기/복제
     ▼
[Step 1] → [Step 2 템플릿+디자인 커스터마이징] → [Step 3 배치/인라인편집] → 인쇄/복사/HTML다운로드
                                                        │ 임시저장 (upsert)
                                                        ▼
buildHtml(): generateNewsletterHTML(...) → 후처리 색상/폰트 치환 → 최종 HTML
```

## 2. DB Design — `bf.newsletters`

### 2.1 마이그레이션 (`supabase-migration-newsletters.sql` 신규)

> 주의: bf 스키마 전환(커밋 014b8bb) 이후이므로 기존 마이그레이션의 `public.` 패턴이 아닌 `bf.`로 작성

```sql
CREATE TABLE IF NOT EXISTS bf.newsletters (
    id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id   uuid REFERENCES bf.projects(id) ON DELETE SET NULL,
    title        text NOT NULL DEFAULT '',
    issue_number text DEFAULT '',
    template     text NOT NULL DEFAULT 'modern',
    config       jsonb NOT NULL DEFAULT '{}',   -- newsletterConfig + custom(색상/폰트)
    sections     jsonb NOT NULL DEFAULT '[]',   -- orderedSections 스냅샷 (items 포함)
    assets       jsonb NOT NULL DEFAULT '{}',   -- boardImageUrls, rewrittenContents, galleryThumbUrls, itemLinks, editedItems
    selection    jsonb NOT NULL DEFAULT '{}',   -- { scheduleIds:[], boardIds:[], galleryIds:[] } — Step 1 복원용
    status       text NOT NULL DEFAULT 'draft' CHECK (status IN ('draft','published')),
    created_at   timestamptz NOT NULL DEFAULT now(),
    updated_at   timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE bf.newsletters ENABLE ROW LEVEL SECURITY;
-- 기존 테이블과 동일한 공개 정책 4종 (select/insert/update/delete USING(true))
CREATE INDEX IF NOT EXISTS idx_newsletters_updated ON bf.newsletters (updated_at DESC);
CREATE INDEX IF NOT EXISTS idx_newsletters_project ON bf.newsletters (project_id);
```

### 2.2 직렬화 방식: 스냅샷 (Plan 쟁점 확정)

| 항목 | 결정 | 근거 |
|------|------|------|
| sections | **스냅샷** (items 통째 저장) | 원본 게시글 수정·삭제와 무관하게 발행본 보존. 게시글 content 포함해도 수십 KB 수준 |
| 이미지 | **URL만 저장** (base64 금지) | public URL이라 만료 없음. 원본 attachment 삭제 시 깨짐은 수용 (문서화) |
| selection | ID 배열 별도 저장 | 불러오기 후 Step 1 복귀 시 체크박스 복원 (삭제된 항목은 자동 무시) |
| custom (색상/폰트) | config.custom에 포함 | 별도 컬럼 불필요 |

## 4. F-1 저장·불러오기 설계

### 4.1 선행 수정: syncSections (재빌드 → 병합)

`goToStep(3)`의 무조건 재빌드를 병합 방식으로 교체. **저장/불러오기 정합성의 전제조건**이며 기존 "Step 2 왕복 시 배치 소실" 결함도 함께 해결:

```javascript
const syncSections = () => {
    const fresh = buildSections();                       // 현재 선택 기준 최신 섹션
    setOrderedSections(prev => {
        if (prev.length === 0) return fresh;
        // 기존 섹션 순서·라벨 유지, 아이템은 (기존 순서 유지 + 신규 선택 추가 + 해제 제거)
        const merged = prev.map(sec => {
            const f = fresh.find(x => x.type === sec.type);
            if (!f) return null;                          // 전부 해제된 섹션 제거
            const freshIds = new Set(f.items.map(i => i.id));
            const kept = sec.items.filter(i => freshIds.has(i.id));
            const keptIds = new Set(kept.map(i => i.id));
            return { ...sec, items: [...kept, ...f.items.filter(i => !keptIds.has(i.id))] };
        }).filter(Boolean);
        // 새로 생긴 타입 섹션은 뒤에 추가
        fresh.forEach(f => { if (!merged.some(s => s.type === f.type)) merged.push(f); });
        return merged;
    });
};
// goToStep: if (step === 3) syncSections();
```

**목록 UI**: 뉴스레터 페이지 Step 1 상단에 접이식 card — `select('id, title, issue_number, template, status, updated_at')` (JSONB 컬럼 제외로 목록 로드 경량화), `updated_at DESC`:

```

┌─────────────────────────────────────────────────────┐
│ 제3호 · 청년노동자인권센터 뉴스레터  [draft]  07-01  │
│                          [불러오기] [복제] [삭제]     │
└─────────────────────────────────────────────────────┘
```

**불러오기** `handleLoadNewsletter(id)`: 단건 `select('*')` → 전체 상태 복원 → `setNewsletterStep(3)` (스냅샷이 있으므로 바로 편집 화면). Step 1 복귀 시 selection 기반 체크박스 유지, `syncSections()` 병합이 스냅샷 순서·라벨 보존.

**복제** `handleDuplicateNewsletter(id)`: 원본 로드 → `issue_number: ''`, `publishDate: 오늘`, `status: 'draft'`로 INSERT → 새 id로 불러오기. "지난 호 복제 → 콘텐츠 교체" 워크플로의 핵심.

**삭제**: `confirm()` 후 DELETE, `currentNewsletterId`가 삭제 대상이면 null로 초기화.

### 4.4 발행 처리 (F-1-7)

인쇄/복사/다운로드 실행 시: `currentNewsletterId`가 있고 status가 draft면 `confirm('이 뉴스레터를 발행 완료로 표시할까요?')` → status='published' UPDATE. 저장 안 된 상태면 묻지 않음 (발행 표시는 아카이브 목적, 강제하지 않음).

## 5. F-2 디자인 커스터마이징 설계

### 5.1 색상 — 출력 HTML 후처리 치환

테마 렌더 함수 수정 없이 `buildHtml()`에서 최종 HTML의 hex를 치환:

```javascript
// 구현: renderNewsletter 클로저 내부 — template/custom을 클로저로 참조 (시그니처 (html))
const applyCustomDesign = (html) => {
    const theme = NL_THEMES[newsletterTemplate] || NL_THEMES.modern;
    const c = newsletterCustom;
    if (c.accent && c.accent !== theme.accent) html = html.split(theme.accent).join(c.accent);
    if (c.accent2 && c.accent2 !== theme.accent2) html = html.split(theme.accent2).join(c.accent2);
    if (c.font) { /* 폰트도 출력 치환 — §5.2 */ }
    return html;
};
const buildHtml = () => applyCustomDesign(
    generateNewsletterHTML(newsletterTemplate, newsletterConfig, applyItemEdits(orderedSections), effectiveOrgName, nlAssets),
    newsletterTemplate, newsletterCustom
);
```

**치환 가능 근거** (코드 확인 완료):
- NL_THEMES 선언 accent가 렌더 출력의 하드코딩 hex와 일치: modern `#0d9488`/`#06b6d4`, magazine `#e85d3a`(3곳), classic `#1e3a5f`(6곳)
- email: `accentColor='#27ae60'` 로컬 상수 → 변수 보간이므로 출력 HTML 치환이 전부 커버
- grid: `#d35400` 인라인 — 동일하게 커버. 단 grid의 카테고리 배지색(`#3498db` 등)은 의도적으로 치환 제외 (주 색상만 커스터마이징)
- `#0d948815` 같은 alpha 접미사 hex도 7자리 prefix 치환으로 유효한 8자리 hex 유지
- **회귀 보장**: custom이 null이거나 기본값과 같으면 치환 스킵 → 기존 출력과 바이트 단위 동일

### 5.2 폰트 — 2계층 적용

```javascript
const NL_FONTS = [
    { id: null,    name: '템플릿 기본 (고딕)' },
    { id: 'serif', name: '명조', stack: "'Noto Serif KR', 'Nanum Myeongjo', 'Batang', serif" },
    { id: 'system', name: '시스템 고딕', stack: "-apple-system, 'Malgun Gothic', sans-serif" },
];
```

**구현 확정: 전 템플릿 출력 치환으로 통일** (설계 초안의 2계층 방식보다 단순, 결과 동일)
- `html.split(theme.font).join(stack)` — themed/grid는 `${theme.font}` 보간 문자열이 출력에 그대로 등장하므로 치환이 커버
- `html.split(NL_EMAIL_FONT).join(stack)` — email 생성기의 더블쿼트 리터럴(`"Pretendard", "Apple SD Gothic Neo", …`)은 별도 상수(NL_EMAIL_FONT)로 치환
- 명조는 웹폰트가 아닌 시스템 폴백 위주 스택이므로 이메일 클라이언트 호환 유지

## 6. F-3 편집 UX 설계

### 6.1 인라인 텍스트 편집 (F-3-1)

**오버라이드 계층** (원본 게시글 불변):

| 대상 | 저장 위치 | 적용 지점 |
|------|-----------|----------|
| 아이템 제목 (전 타입) | `editedItems[id].title` | `applyItemEdits()` — sections 복사본에 적용 |
| 게시글 본문 | `rewrittenContents[id]` (기존 재사용) | `_nlRenderSections` 기존 로직 그대로 |

```javascript
const applyItemEdits = (sections) => sections.map(sec => ({
    ...sec,
    items: sec.items.map(it => editedItems[it.id]?.title !== undefined
        ? { ...it, title: editedItems[it.id].title, name: editedItems[it.id].title }  // gallery는 name 사용
        : it),
}));
```

**UI**: Step 3 아이템 행에 ✏️ 버튼 추가 (기존 🔗 옆) → 클릭 시 행 아래 편집 영역 확장:
- 제목 input (초기값: 현재 표시 제목)
- 게시글이면 본문 textarea (초기값: `rewrittenContents[id] || stripHtml(content)` — AI 재작성 결과 이어서 편집 가능)
- [원본으로 되돌리기] 버튼 → editedItems/rewrittenContents에서 해당 id 삭제

### 6.2 섹션 라벨 편집 (F-3-2)

- Step 3 섹션 헤더의 라벨 텍스트를 클릭 시 input으로 전환 (`orderedSections[si].label` 직접 수정)
- **렌더러 수정**: `_nlRenderSections`의 하드코딩 라벨을 `sec.label` 우선으로 변경 + **XSS 방지 이스케이프** (사용자 자유 입력이므로 필수 — Check 단계에서 발견·수정):
  ```javascript
  theme.sectionLabel(esc(sec.label || '주요 일정'))   // 기존: theme.sectionLabel('주요 일정')
  ```
- **email/grid**: 시그니처에 `labels = {}` 파라미터 추가, 디스패처에서 `{ schedules: s?.label, boards: b?.label, gallery: g?.label }` 전달. 생성기 내부 하드코딩 '주요 소식'/'주요 일정'을 `labels.boards || '주요 소식'` 폴백으로 교체 → 기본 동작 불변

### 6.3 HTML 파일 다운로드 (F-3-3)

```javascript
const handleDownloadOrdered = () => {
    if (orderedSections.length === 0) { alert('콘텐츠가 없습니다.'); return; }
    const blob = new Blob([buildHtml()], { type: 'text/html' });
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `뉴스레터_${newsletterConfig.issueNumber || newsletterConfig.publishDate}.html`;

};
```

### 6.4 미리보기 모드 토글 (F-3-4)

```

```

| 모드 | iframe 스타일 | 컨테이너 |
|------|--------------|----------|
| desktop (기존) | width:200%, scale(0.5), origin top-left | height:700, **overflow-y:auto로 변경** (긴 뉴스레터 스크롤 — 기존 1400px 클리핑 개선) |

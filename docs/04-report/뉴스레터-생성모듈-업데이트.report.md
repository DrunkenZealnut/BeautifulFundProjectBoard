## 3. 주요 기술 결정

1. **스냅샷 직렬화** — orderedSections를 통째 저장(items 포함)하여 원본 게시글 수정·삭제와 무관하게 발행본 보존. 이미지는 public URL만 저장(만료 없음).
2. **출력 HTML 후처리 치환** — 테마 함수들이 `theme.accent`를 참조하지 않고 hex를 하드코딩하는 구조를 활용, 함수 전면 수정 대신 `split().join()` 치환으로 6종 템플릿 일괄 커스터마이징. 미사용 시 출력 불변(회귀 없음).
3. **오버라이드 슬롯 재사용** — 본문 인라인 편집을 AI 재작성(`rewrittenContents`)과 동일 슬롯에 저장, 신규 상태 최소화 + 원본 게시글 불변 보장.
4. **syncSections 병합** — Step 3 진입 시 무조건 재빌드하던 기존 로직을 병합 방식으로 교체, 저장/불러오기 정합성 확보 + "Step 왕복 시 배치 소실" 결함 해결.

## 5. 학습 포인트

- **설계 전 코드 grep이 핵심 결정을 좌우** — "테마 함수가 accent를 참조하는가"를 grep(0건)으로 확인한 것이 출력 치환 아키텍처 결정의 근거가 됨. 추측 대신 검증.
- **gap-detector가 이스케이프 정책 불일치를 포착** — email/grid는 `esc()` 적용, themed만 무방비였던 것을 교차 비교로 발견. 단일 사용자 self-XSS라 위험은 낮으나 일관성 결함을 조기 차단.
- **verify 스킬의 런타임 검증** — 구문 통과만으로는 알 수 없는 색상 치환 완전성(#7c3aed 0회)·편집 반영을 실제 브라우저 주행으로 확인.
